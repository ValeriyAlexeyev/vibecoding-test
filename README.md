# Вебкодинг: от новичка до синьора

Локальное обучающее веб-приложение по веб-разработке. Работает без платных API, подписок, облачных сервисов и внешних баз данных. Уроки и учебные данные лежат в проекте, а личный прогресс сохраняется в `localStorage` браузера.

## Что внутри

- Главная страница с общей картой обучения.
- Дорожная карта: Новичок, Junior, Middle, Senior.
- Уроки с фильтром по уровню и поиском.
- Карточки уроков с теорией, примером кода, практикой и тестом.
- Отметка урока как пройденного.
- Тесты с вариантами ответов и подсчетом результата.
- Практические задания и мини-проекты.
- Личный прогресс и шкала выполнения.
- Справочник HTML/CSS/JS.
- Финальные проекты и раздел «Путь до Senior».
- Мини-редактор HTML/CSS/JS с предпросмотром.
- Локальный AI-агент без внешних API.
- Светлая и темная тема.

## Как запустить

### Быстрый запуск на этом компьютере

Откройте файл `start-site.cmd` двойным кликом. Он запустит локальный сервер и покажет адрес:

```text
http://127.0.0.1:5173/
```

Важно: сайт открывается по этому адресу только пока запущено окно сервера.

### Стандартный запуск

Установите зависимости:

```bash
npm install
```

Запустите локальный сервер:

```bash
npm run dev
```

После запуска откройте адрес, который покажет Vite, обычно это `http://127.0.0.1:5173`.

Для production-сборки:

```bash
npm run build
```

## Структура проекта

```text
index.html
package.json
src/
  main.jsx
  components/
  data/
    course.js
  styles/
    main.css
README.md
```

`src/main.jsx` содержит React-компоненты приложения и работу с состоянием.  
`src/data/course.js` содержит локальные учебные данные: уровни, уроки, практику, мини-проекты, справочник и финальные проекты.  
`src/styles/main.css` содержит темы, адаптивность и визуальную систему.

## Как добавлять новые уроки

Откройте `src/data/course.js` и добавьте объект в массив `lessons`.

Минимальная структура урока:

```js
{
  id: "new-lesson-id",
  title: "Название урока",
  level: "beginner",
  duration: "45 мин",
  description: "Краткое описание",
  theory: "Теория урока",
  code: "console.log('пример');",
  practice: "Практическое задание",
  quiz: []
}
```

Доступные уровни: `beginner`, `junior`, `middle`, `senior`.

## Как добавлять тесты

Внутри урока заполните массив `quiz`.

```js
quiz: [
  {
    question: "Что делает HTML?",
    options: ["Описывает структуру", "Хранит базу данных", "Покупает домен"],
    answer: 0
  }
]
```

`answer` хранит индекс правильного ответа в массиве `options`. Индексация начинается с нуля.

## Как работает сохранение прогресса

Прогресс хранится в `localStorage` по ключу:

```text
webcoding-platform-state
```

Туда сохраняются:

- пройденные уроки;
- выполненные практические задания;
- результаты тестов;
- код из мини-редактора;
- выбранная тема.
- история сообщений AI-агента.

База данных и сервер не используются. Если очистить данные сайта в браузере, прогресс сбросится.

## Как работает AI-агент

AI-агент находится в разделе `AI-агент`. Он не использует OpenAI API, внешние модели, серверы или облачные сервисы. Ответы формируются локально в браузере на основе:

- данных уроков из `src/data/course.js`;
- текущего прогресса пользователя;
- выполненных практических заданий;
- кода из мини-редактора;
- набора правил в `src/main.jsx`.

Агент умеет:

- подсказать следующий урок;
- объяснить тему из курса;
- предложить практическое задание;
- направить к тесту;
- проверить код по базовым правилам: семантика, заголовки, кнопки, `alt`, `localStorage`, `flex/grid`.

История чата сохраняется в `localStorage` вместе с прогрессом.

## Как доработать проект дальше

- Вынести крупные React-компоненты из `src/main.jsx` в отдельные файлы внутри `src/components/`.
- Перенести уроки из `course.js` в отдельные JSON-файлы, например `lessons.json`, `projects.json`, `reference.json`.
- Добавить маршрутизацию для прямых ссылок на уроки.
- Добавить импорт и экспорт прогресса в JSON-файл.
- Добавить больше практических задач и финальных проектов.
- Добавить режим code review с чек-листом качества.
- Расширить AI-агента: вынести правила в отдельный файл, добавить больше проверок кода и персональные рекомендации по слабым темам.
- Добавить простые unit-тесты для функций прогресса и проверки тестов.
